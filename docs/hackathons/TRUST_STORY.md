# Trust Story for Judges

## What trust output Ghost Chimera produces

- Policy verdicts and deny reasons
- Confidence-bearing result envelopes
- Verification outcomes and warning surfaces
- Replayable telemetry and run summaries
- Production readiness checks

## Demo trust panel checklist

- [ ] Show policy gate behavior on restricted action
- [ ] Show a successful governed run with confidence and provenance
- [ ] Show readiness endpoint (`/api/console/readiness`)
- [ ] Show clear boundary: beta runtime, not AGI, no untrusted host execution

## Security findings and governance framing

Use `SECURITY.md`, production mode checks, and readiness outputs as first-class demo artifacts, not only implementation details.

## Evidence package expected after a run

- Objective and compiled task context
- Decision/scheduler trail summary
- Verification and confidence summary
- Policy outcomes
- Suggested PR/change package
